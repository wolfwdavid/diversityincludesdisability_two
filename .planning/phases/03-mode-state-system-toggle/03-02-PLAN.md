<objective>
Land the runtime mode substrate: (1) the Svelte 5 runes store that mirrors the pre-paint DOM `data-mode`
and writes the user's choice through to the namespaced `did2:mode` key, and (2) the synchronous inline
`<head>` script in `app.html` that stamps `data-mode` before first paint (the ONLY correct anti-FOUC method
on adapter-static, which has no SSR request). CSS adds the `.visually-hidden` util, `[data-mode]` hooks, and
the no-transition-until-ready guard. A `parity.spec.ts` guards the inline-script/resolver drift risk.

Purpose: Satisfies MODE-04 (no flash) and the persistence half of MODE-01/02. The store is what the
Plan-03 toggle component drives; the inline script is what makes the resolved mode exist on first paint.

      // SSR/prerender default = 'accessible' (zero-WebGL baseline if JS fails).
      // On client, SEED from the attribute the inline script already set — do NOT re-resolve.
      let current = $state<Mode>('accessible');
      if (browser) {

        if (stamped === 'premium' || stamped === 'accessible') current = stamped;

      export function getMode(): Mode { return current; }

      export function setMode(next: Mode): void {
